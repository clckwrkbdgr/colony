СЦЕНАРИЙ:

Сценарий представляет собой папку, в которой хранится файл сценария и другие.
Файл сценария имеет имя "scenario". Все пути в нем и дочерних файлах (например,
профилей) есть относительны к папке сценария.
Рекомендуется ради удобства хранить все файлы, относящиеся к данному сценарию,
в одной папке с ним.

ФОРМАТ ФАЙЛА СЦЕНАРИЯ:

Файл сценария - обычный текстовый файл, каждая строка которого рассматривается
как инструкция.
Первой непустой строкой должно быть объявление спрайта пола:
	floor <file_name>
Здесь file_name - имя файла спрайта (графический файл).
Дальше идут остальные команды.
Каждая инстанция характеризуется файлом профиля и количеством создаваемых
особей:
	instance <profile_file_name> <amount>
Обязательно должно быть описание карты (ее размеров):
	map <width> <height>

ФОРМАТ ФАЙЛА ПРОФИЛЯ:

Файл сценария - обычный текстовый файл, каждая строка которого рассматривается
как инструкция.
Первой строкой должно быть объявление имени класса (см. ФОРМАТ ФАЙЛА СЦЕНАРИЯ).
Класс задается одним из значений из списка: {creature,plant,animal}.
Дальше идут строки свойств. Каждое свойство имеет вид:
	<name> <value>
Свойство, не указанное в файле профиля, будет иметь значение по умолчанию.
Для свойств, инкапсулирующих спрайты, значение должно быть именем файла.
Остальные свойства подразумеваются целыми числами.
Свойства с неизвестными именами будут просто пропускаться.

СВОЙСТВА ПРОФИЛЕЙ:

Профиль Существа имеет три свойства - продолжительность жизни (maxAge), и два
спрайта - живой (aliveSprite) и мертвый (deadSprite).
Продолжительность жизни по умолчанию равна 1, спрайты имеют индекс -1 - т.е.
никакого спрайта (пустое изображение).

Профиль Растения наследует свойства Существа плюс масса (mass), по
умолчанию равна 1.

Профиль Животного наследует свойства Существа плюс две группы
параметров для сна и голода - количество циклов, нужных для того, чтобы животное
ощутило себя сонным/голодным и кол-во, нужное для того, чтобы Существо умерло от
недосыпа/голода (соотв. getSleepy, getHungry, deadSleepy, deadHungry).
По умолчанию несмертельные свойства имеют значение 0 - т.е. Животное вечно
сонное и голодное, а смертельные - по 1 (т.е. на следующем после старта ходу
Животное умрет от недосыпа/голода).

СПРАЙТЫ:

Спрайт - изображение размером 32х32.
Все спрайты объектов (это не касается спрайтов пола) обладают прозрачностью.
Места, которые на спрайте должны быть прозрачны, закрашиваются цветом
прозрачности, который определяется по цвету одного из углов изображения (чаще
всего - верхний левый).

БАГИ:

Единственный известный баг - непредсказуемое поведение Существ, созданных
копи-конструктором или оператором присваивания (вплоть до обрушения программы).
